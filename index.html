How OpenNewt Works

A continuous 4-phase cycle inspired by biological neural regeneration.

1

Sense — Damage Detection

Somatic layer performs continuous heartbeat probes, AST health scans, and environment monitoring. Detects anomalies in < 5 seconds.

Soma · Perception Layer
2

Decide — Strategy Generation

Neural Core generates multiple repair strategies via LLM sampling, evaluates each with a plasticity assessment matrix, and picks the optimal path.

Plasticus · Decision Layer
3

Act — Precision Repair

Effector executes repairs through micro/local/full regeneration modes. Supports dry run simulation and gradual cutover for zero-downtime recovery.

Effector · Execution Layer
4

Remember — Immune Learning

Mnemosyne records every repair event, builds a code graph, and develops immune memory. Future similar failures are resolved faster.

Mnemosyne · Memory Layer

Get Started

Up and Running in 2 Minutes

Install, configure, and launch your first self-healing scan.

bash 
# Clone & install
git clone https://github.com/Axonewt/opennewt.git
cd opennewt
pip install -r requirements.txt

# Configure LLM (Ollama local or API key)
cp config.example.yaml config.yaml
# Edit config.yaml → llm.provider: ollama

# Launch API server + Dashboard
python api_server.py --port 8088
python — Register a target 
import httpx

# Register your codebase
resp = httpx.post("http://127.0.0.1:8088/api/targets", json={
    "name": "my-quant-system",
    "path": "/opt/quant-trading",
    "auto_heal": True,
    "health_threshold": 0.7
})

# Or use MCP in Claude Desktop
# → mcp config add opennewt
# → "Scan my quant system"
